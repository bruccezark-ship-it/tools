# 文件保存技能 (file-saver)

## 描述

本技能用于将用户在 TRAE 对话框中上传的文件、图片等内容保存到当前项目的目录中，使项目可以直接使用这些文件。

## 何时使用

**使用此技能当：**
- 用户在对话框中上传了文件或图片，并希望保存到项目中
- 用户明确要求"保存这个文件"、"把图片存到项目里"等类似操作
- 用户上传文件后询问如何在项目中使用
- 用户提到需要将对话中的附件持久化存储

**不要使用此技能当：**
- 用户只是上传文件作为上下文参考，没有明确要求保存
- 文件已经存在于项目目录中（通过 # 引用的文件）
- 用户只是想查看文件内容而不需要保存

## 核心概念

### 默认保存路径
- 主目录：`assets/uploads/`
- 图片文件：`assets/uploads/images/`
- 文档文件：`assets/uploads/docs/`
- 其他文件：`assets/uploads/other/`

### 支持的文件类型
- **图片类**: jpg, jpeg, png, gif, svg, webp, bmp, ico
- **文档类**: pdf, doc, docx, txt, md, csv, xls, xlsx, json, yaml, yml, xml
- **代码类**: js, ts, py, java, go, rs, cpp, c, h, html, css
- **其他**: 所有其他类型文件

## 指令

### 工作流程

当检测到用户上传文件并需要保存时，按以下步骤执行：

1. **确认保存意图**
   - 如果用户明确要求保存，直接执行
   - 如果用户只是上传但未明确，询问是否需要保存到项目中
   - 确认保存位置（使用默认路径或用户指定路径）

2. **识别文件信息**
   - 文件名（保留原始文件名）
   - 文件类型（根据扩展名判断）
   - 文件内容（文本文件可直接获取内容，图片需特殊处理）

3. **创建目录结构**
   - 检查 `assets/uploads/` 目录是否存在
   - 不存在则创建相应的子目录

4. **保存文件**
   - 文本文件：直接写入文件内容
   - 图片/二进制文件：说明保存方式并提供指导

5. **验证与反馈**
   - 确认文件已保存
   - 告知用户文件路径
   - 说明如何在项目中使用该文件

### 文本文件保存步骤

对于文本类文件（.txt, .md, .json, .csv, .xml, .yaml, 代码文件等）：

1. 从对话上下文中提取文件内容
2. 确定保存路径和文件名
3. 使用文件写入工具保存文件
4. 验证文件是否成功创建

### 图片文件处理

对于图片文件，按以下方式处理：

1. 确认图片文件名和格式
2. 在 `assets/uploads/images/` 目录下创建占位说明
3. 提供保存图片的具体指导：
   - 说明用户可以手动将图片保存到项目目录
   - 提供推荐的目录结构
   - 说明如何在代码中引用这些图片

### 批量文件处理

如果用户上传多个文件：
1. 逐个确认文件类型和内容
2. 按类型分别保存到对应子目录
3. 最后汇总所有保存的文件路径

## 使用示例

### 示例 1：保存上传的文本文件

用户上传了 `config.json` 文件并说"帮我把这个配置文件存到项目里"

执行步骤：
1. 读取对话上下文中的 config.json 内容
2. 创建 `assets/uploads/` 目录（如果不存在）
3. 将内容写入 `assets/uploads/config.json`
4. 告知用户文件已保存到 `assets/uploads/config.json`

### 示例 2：保存图片

用户上传了一张截图并说"把这张图片保存到项目中"

执行步骤：
1. 确认图片文件名（如 screenshot.png）
2. 创建 `assets/uploads/images/` 目录说明
3. 告知用户：
   - 图片文件建议手动保存到 `assets/uploads/images/` 目录
   - 可以通过拖拽或复制粘贴到项目文件树
   - 在代码中可以通过相对路径引用

## 目录结构规范

```
项目根目录/
├── assets/
│   └── uploads/          # 上传文件保存根目录
│       ├── images/       # 图片文件
│       ├── docs/         # 文档文件 (pdf, doc, txt 等)
│       └── other/        # 其他类型文件
```

## 注意事项

1. **文件命名**：尽量保留原始文件名，如有重名则添加时间戳后缀
2. **路径安全**：确保保存路径在当前项目目录内，不要写入系统目录
3. **大文件处理**：对于特别大的文件，提醒用户注意项目大小
4. **隐私提醒**：提醒用户不要上传包含敏感信息的文件
5. **Git 忽略**：建议用户根据需要将 `assets/uploads/` 添加到 .gitignore

## 失败处理

- 如果文件内容无法获取：告知用户当前限制，提供手动保存的指导
- 如果目录创建失败：检查权限并告知用户手动创建
- 如果文件写入失败：检查路径和权限，提供替代方案
